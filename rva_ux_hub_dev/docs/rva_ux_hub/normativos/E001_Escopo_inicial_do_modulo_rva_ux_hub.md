## Objetivo
Definir o escopo inicial pratico do modulo `rva_ux_hub` na fase de bootstrap
documental e estrutural.

## Escopo desta fase

O modulo deve nascer com:

- guia principal conversacional
- selecao manual de rota baseada em arquivo oficial
- acao de `help` para explicar o roteamento e a manutencao
- guia `trade` criada como estrutura inicial de navegacao
- preparacao para crescimento futuro de novas guias e rotas

## Fora de escopo nesta fase

- UX final do produto
- painel completo de acompanhamento
- cards operacionais completos por modulo
- `GSOL` como orquestrador ativo
- roteamento automatico por aprendizado
- seletores avancados de modelo expostos ao usuario final

## Rotas iniciais ativas

As rotas ativas nesta fase devem refletir apenas o que ja pode ser sustentado
com coerencia pelo projeto.

Direcao inicial:

- `geral`
- `trade`

Direcao futura orientada pelo mesmo contrato:

- `financeiro`
- `programming`
- `marketing`

## Regra de crescimento

Novas rotas ou novas guias especializadas so devem entrar quando:

- houver entrada correspondente no registry oficial
- houver contrato minimo do modulo correspondente
- a governanca documental do modulo ja estiver oficializada
